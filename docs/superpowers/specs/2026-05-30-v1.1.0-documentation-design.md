# v1.1.0 Documentation Update — Design Spec

**Date:** 2026-05-30
**Scope:** Approach B — Full refresh for the v1.1.0 release

## Context

24 commits since v1.0.1 introduce significant new features that are not yet reflected in the
README, CHANGELOG, or `docs/`. This spec covers all documentation changes needed before tagging v1.1.0.

## 1. CHANGELOG.md — `[1.1.0]` section

Add a new section above `[1.0.1]`. Grouped by category:

### Added

- Recommendation engine "What to Play Tonight" (`/play-tonight`) — content-based algorithm using playtime history, taste profile, session engagement, and user ratings
- IGDB lazy matching to enrich game metadata for recommendations (optional, configurable in Settings)
- Taste Profile page (`/profile`) — inferred genre/system preferences computed in background, with transparency metrics and quality score
- Post-session rating prompts — users can rate sessions; ratings feed back into future recommendations
- Session engagement classification — sessions categorised by engagement level (each session carries an `engagementLevel` field)
- `pnpm dev:all` — launches Next.js and scrobbler together (replaces running two terminals)
- `pnpm dev:all:mobile` — same, bound to `0.0.0.0` for local network access
- Inherited userdata sessions separated from scrobbler-recorded sessions in the DB schema
- Now Playing shows EmulationStation browsing state and screensaver mode (not just active game)
- English translations for Profile page and IGDB settings

### Fixed

- PWA: `apple-mobile-web-app-capable` meta tag missing on iOS Safari
- PWA: reliability and UX improvements (install flow, update detection)
- Stats: NULL-safe `recalboxId` comparison using `IS` operator for total playtime query
- Wrapped: playtime displayed in minutes when under 1 hour
- Wrapped: navbar and theme toggle missing on archive page
- SSH: patron status check failures logged as plain message (no stack trace noise)
- API: error logging and input validation hardened
- API: `logger.debug` replaced with `logger.info` (was undefined at runtime)
- Recommendations: `inArray` with full collection avoided to prevent SQLite variable limit error
- Layout: `mx-auto` and `px-4` applied to Play Tonight and Profile page containers

### Performance

- SSH circuit breaker — when a Recalbox is unreachable, subsequent SSH calls fail fast with a reduced timeout instead of waiting the full connect timeout
